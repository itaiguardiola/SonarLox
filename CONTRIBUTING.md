# Contributing to SonarLox

We welcome contributions to SonarLox! This document outlines how to contribute to the project.

## How to Contribute

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/AmazingFeature`)
3. Commit your changes (`git commit -m 'Add some AmazingFeature'`)
4. Push to the branch (`git push origin feature/AmazingFeature`)
5. Open a Pull Request

## Code Style

- Follow the existing code style
- Write clear, descriptive commit messages
- Add JSDoc documentation for new components and functions
- Ensure all tests pass before submitting

## Reporting Issues

- Use the issue tracker to report bugs
- Include detailed steps to reproduce the issue
- Provide system information and versions

## Pull Request Process

1. Ensure any install or build dependencies are removed before the end of the layer when doing a build
2. Update the README.md with details of changes to the interface
3. Increase the version numbers in any examples files and the README.md to the new version that this Pull Request would represent
4. The version number should follow semantic versioning
